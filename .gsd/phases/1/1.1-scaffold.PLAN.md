# Plan 1.1: Project Scaffold & UI Foundation

## Objective

Initialize the Next.js application, configure the styling system (Tailwind + Shadcn) with the required "Claymorphism" aesthetic, and establish the core layout structure.

## Context

- .gsd/SPEC.md (Design Requirements)
- .gsd/phases/1/RESEARCH.md (Stack decisions)

## Tasks

<task type="auto">
  <name>Initialize Next.js App</name>
  <files>
    package.json
    tsconfig.json
    next.config.js
    components.json
  </files>
  <action>
    Initialize a new Next.js 14 project with TypeScript, Tailwind CSS, and ESLint.
    - Run `npx create-next-app@latest . --typescript --tailwind --eslint` (if dir empty) or strictly follow existing structure.
    - Initialize shadcn-ui: `npx shadcn-ui@latest init` (Style: Default, Base Color: Slate, CSS Variables: Yes).
    - Ensure directory structure: `app/`, `components/`, `lib/`.
  </action>
  <verify>
    Run `npm run dev` and check http://localhost:3000 returns 200 via curl or browser tool.
  </verify>
  <done>
    Next.js default page loads without errors. `components.json` exists.
  </done>
</task>

<task type="auto">
  <name>Configure Claymorphism Theme</name>
  <files>
    tailwind.config.ts
    app/globals.css
  </files>
  <action>
    Implement Custom Claymorphism Utilities in Tailwind.
    - Extend `tailwind.config.ts` to add drop-shadows and border-radius utilities specific to claymorphism.
    - Add `.clay-card` and `.clay-btn` utility classes in `app/globals.css` or plugin.
    - Configure Light/Dark mode support (using `next-themes`).
    - Update `app/layout.tsx` to include `ThemeProvider`.
  </action>
  <verify>
    Create a test page `app/test-theme/page.tsx` with a `.clay-card` element.
    Take a screenshot or check computed styles.
  </verify>
  <done>
    Claymorphism styles are applied correctly. Dark mode toggle works (verifiable via class change on html tag).
  </done>
</task>

<task type="auto">
