# Phase 10: Mobile Responsiveness

## Goal

Ensure the application is fully functional and visually usable on mobile devices and tablets.

## Problem

- **Sidebar**: Currently `hidden md:flex`, meaning mobile users have no navigation.
- **Tables**: Shadcn tables without a scroll wrapper break layout on small screens.
- **Padding**: `p-10` in settings layout might be too large for mobile.

## Proposed Changes

### 1. Mobile Sidebar

- **Component**: Use `Sheet` (from Shadcn/Radix) to create a slide-out sidebar.
- **Trigger**: Add a Hamburger Menu button in the `Header` or top-left of the layout (visible only on mobile).
- **Implementation**: Refactor `Sidebar` to be reusable or wrap it in the Sheet.

### 2. Layout Adjustments

- **Root Layout**: Ensure `p-6 md:p-8` is responsive.
- **Settings Layout**: Change `p-10` to `p-4 md:p-10`.
- **Tables**: Wrap all `Table` components in a `div` with `overflow-x-auto` (Shadcn `Table` usually does this, but we need to verify).

### 3. Component Fixes

- **Dialogs**: Ensure dialogs fit on small screens (max-w-[90vw]).
- **Cards**: Ensure grids collapse to 1 column on mobile (`grid-cols-1 md:grid-cols-2`).

## Detailed Plan

### Wave 1: Mobile Navigation

- [ ] Install `Sheet` component (`npx shadcn-ui@latest add sheet`).
- [ ] Modify `components/layout/sidebar.tsx` to export a `MobileSidebar` component.
- [ ] Update `components/layout/header.tsx` (or `app/(protected)/layout.tsx`) to include the Mobile Sidebar trigger.

### Wave 2: Layout & Tables

- [ ] Review `app/(protected)/settings/layout.tsx` padding.
- [ ] Check `property-list.tsx` and `complaint-list.tsx` for table overflow.
